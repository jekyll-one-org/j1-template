[role="dropcap"]
PhotoSwipe Lightbox is a JavaScript library to create lightboxes placed on
images in webpages. When you click such images, they open in a new,
overlaying layer, darkening the background and focusing the image.

[role="mt-5"]
== Auto initialization

Adds Lightbox V3-style auto-initialization to PhotoSwipeLightbox so
that any element matching the configurable selector (default
`[data-photoswipe-lightbox]`) is treated as a gallery container
without requiring callers to instantiate `new PhotoSwipeLightbox(...)`
and `.init()` themselves.

[role="mt-5"]
The element carrying `data-photoswipe-lightbox` is the gallery; its
descendant `<a>` tags are the slides. The attribute value is treated
purely as a logical id (useful for getAutoInitInstance lookups) and
is not required.

Per-gallery overrides via data attributes on the gallery element:
  data-pswp-children   children selector (default: 'a')
  data-pswp-options    JSON object merged on top of the global options
                       (e.g. data-pswp-options='{"bgOpacity":0.9}')

Public static API on PhotoSwipeLightbox:
  PhotoSwipeLightbox.autoInit([opts], [root])     scan & create instances
  PhotoSwipeLightbox.autoInitDestroy()            destroy all auto instances
  PhotoSwipeLightbox.autoInitObserve()            watch DOM for new galleries
  PhotoSwipeLightbox.getAutoInitInstance(elOrSel) look up instance by element
  PhotoSwipeLightbox.autoInitSelector             selector (overridable)
  PhotoSwipeLightbox.autoInitOptions              defaults for autoInit()
  PhotoSwipeLightbox.autoInitDisabled             skip DOMContentLoaded boot

Default behaviour: a DOMContentLoaded listener inside this module
calls autoInit() once with `PhotoSwipeLightbox.autoInitOptions` and
then starts a MutationObserver so dynamically added galleries are
covered too. To suppress the auto-bootstrap (e.g. when a J1 adapter
wants full control of timing), set
  PhotoSwipeLightbox.autoInitDisabled = true;
before DOMContentLoaded fires, then call autoInit() yourself.

Backwards compatibility: the existing `new PhotoSwipeLightbox(opts)`
+ manual `.init()` workflow is unchanged. The extension is purely
additive and only acts on elements matching `autoInitSelector`.

The element carrying `data-photoswipe-lightbox` becomes the gallery container.
Its descendant `<a>` tags are the slides. This mirrors PhotoSwipe's standard
markup convention (gallery + children), which is a better fit than
Lightbox V3's per-link grouping because PhotoSwipe needs an ancestor element
to bind a delegated click listener to. The attribute *value* is treated as
a logical group id, useful only for `getAutoInitInstance` lookups.

// [role="mt-5"]
// === What was added 

// One block, inserted right before `return PhotoSwipeLightbox;`

// - `PhotoSwipeLightbox.autoInit([opts], [root])` — scans for 
//   `[data-photoswipe-lightbox]` and creates a `PhotoSwipeLightbox` instance
//   per match. Returns the new instances. Skips elements already bound, so it's
//   safe to call repeatedly.
// - `PhotoSwipeLightbox.autoInitDestroy()` — destroys every auto-init instance,
//   removes click listeners, and disconnects the DOM observer.
// - `PhotoSwipeLightbox.autoInitObserve()` — starts a debounced (rAF-coalesced) 
//   MutationObserver` that re-runs `autoInit()` whenever new matching elements
//   are inserted (galleries from AJAX, J1 carousels, hot reload ... ).
// - `PhotoSwipeLightbox.getAutoInitInstance(elementOrSelector)` — looks up
//   the instance attached to a gallery element.
// - `PhotoSwipeLightbox.autoInitSelector` / `.autoInitOptions` / `.autoInitDisabled`
//   overridable knobs for selector, default options, and bootstrap suppression.
// - A DOMContentLoaded bootstrap that runs `autoInit()` + `autoInitObserve()`
//   once with the current `autoInitOptions` (skipped if `autoInitDisabled = true`
//   is set first).

// [role="mt-4"]
// Per-gallery overrides::
// Via data attributes on the gallery element:
//   - `data-pswp-children="..."` — children selector (default: `'a'`)
//   - `data-pswp-options='{"bgOpacity":0.7,"showHideAnimationType":"fade"}'`
//     JSON merged on top of the global options for that gallery only

// `pswpModule` resolution:: 
// If `pswpModule` is not in the merged options but `window.PhotoSwipe`
// exists (which `photoswipe-core.js` registers via its UMD wrapper),
// it is used automatically. So loading both UMD scripts in order is enough;
// no manual wiring needed.

// Backward compatibility::
// Purely additive. The existing `new PhotoSwipeLightbox({gallery: '#my-gallery', children: 'a', pswpModule: PhotoSwipe}).init()` workflow you already use in your Swiper template is untouched. The auto-init
// layer only acts on elements matching `autoInitSelector`, which is opt-in
// markup.

// Verified via smoke test::
// That the modification correctly: creates one instance per gallery, attaches
// click listeners to the gallery containers, merges per-gallery JSON options,
// defaults `pswpModule` from `window.PhotoSwipe`, is idempotent on repeated
// calls, and rebuilds cleanly after `autoInitDestroy()`.
