== lightGallery examples

The following demos show how the J1 Template uses lightGallery for
fully responsive image and video galleries. The template is built
on Bootstrap and reuses the standard grid classes
(`col-xs-*`, `col-sm-*`, `col-md-*`) on the slide list to produce
a tiling thumbnail layout *without* any extra CSS.

=== lightGallery on videos

lightGallery supports YouTube, Vimeo, VK, Dailymotion and every
HTML5 video format the browser knows (MP4, WebM, Ogg, ...).

To display a YouTube, Vimeo or VK video, paste the public video URL
(or the *share* URL provided by the platform) into the `data-src`
attribute of the slide – exactly like for an image. lightGallery
inspects the URL, recognises the platform and creates the matching
video slide automatically.

You can also provide a *poster image* per video. The poster is
displayed instead of the video while the user navigates between
slides. Posters keep the gallery responsive (so the user can swipe
or drag to other slides without waiting for the video to load) and
they reduce the total bandwidth needed for the gallery. The video
itself is loaded on demand, when the user clicks the poster. Set
the poster URL through the `data-poster` attribute.

When the *thumbnail* plugin is loaded, lightGallery can also fetch
the YouTube, Vimeo and Dailymotion thumbnails automatically. The
thumbnail size is configured through `youTubeThumbSize` (see the
*Thumbnails Plugin* section in the API reference for the allowed
values). Videos are paused automatically when the user navigates to
another slide – no extra code is needed on your side.

[role="mt-4"]
=== HTML5 video

To display a local HTML5 video, place the `<video>` tag inside a
hidden `<div>` on the page and reference the wrapper through the
`data-html` attribute on the slide. *Do not* set `data-src` for
HTML5 video slides – `data-html` already tells lightGallery what to
render. As an alternative, the HTML markup itself can be passed
directly as the value of `data-html`.

The example below uses the `lg-video-object lg-html5` classes on
the `<video>` element so the *video* plugin can recognise it as an
HTML5 video.

[role="mt-4"]
==== Plugin dependency

HTML5 video slides require the *video* plugin. If you want to use
the *video.js* skin instead of the native browser controls, also
load the video.js library and stylesheet:

[source, html]
----
<!-- video.js (optional, only when `videojs: true`) -->
<link rel="stylesheet" href="https://vjs.zencdn.net/8.10.0/video-js.css" />
<script src="https://vjs.zencdn.net/8.10.0/video.min.js"></script>

<!-- lightGallery video plugin (always required for video slides) -->
<script src="js/plugins/lg-video.min.js"></script>
----

[role="mt-4"]
==== HTML markup

[source, html]
----
<!-- Hidden <video> elements (one per slide) -->
<div style="display:none;" id="video_1">
  <video class="lg-video-object lg-html5" controls preload="none">
    <source src="/path/to/video_1.mp4" type="video/mp4">
    Your browser does not support HTML5 video.
  </video>
</div>
<!-- ... -->
<div style="display:none;" id="video_n">
  <video class="lg-video-object lg-html5" controls preload="none">
    <source src="/path/to/video_n.mp4" type="video/mp4">
    Your browser does not support HTML5 video.
  </video>
</div>

<!-- Gallery markup -->
<div class="j1-lg-galleryblock">
  <div class="title">Video gallery title</div>

  <ul id="j1-lg-video-html5-ul" class="row j1-lg-gutter list-unstyled">

    <li class="col-xs-6 col-sm-4 col-md-3 video"
        data-html="#video_1"
        data-poster="/path/to/video_1-poster.jpg"
        data-sub-html="Caption text for video_1">
      <a class="thumbnail" href="#">
        <img class="img-fluid" src="/path/to/video_1-thumbnail.jpg">
      </a>
    </li>

    <!-- ... -->

    <li class="col-xs-6 col-sm-4 col-md-3 video"
        data-html="#video_n"
        data-poster="/path/to/video_n-poster.jpg"
        data-sub-html="Caption text for video_n">
      <a class="thumbnail" href="#">
        <img class="img-fluid" src="/path/to/video_n-thumbnail.jpg">
      </a>
    </li>

  </ul>
</div>

<!-- Initialisation (v2 API) -->
<script>
  const galleryEl = document.getElementById('j1-lg-video-html5-ul');
  lightGallery(galleryEl, {
    selector: 'li',
    plugins:  [lgVideo, lgThumbnail],
    videojs:  true
  });
</script>
----

=== Online video

Online video slides are even simpler than HTML5 video slides:
because YouTube, Vimeo, VK and Dailymotion provide their own player,
no hidden `<video>` element is needed. Set `data-src` to the public
video URL (or the *share* URL) – lightGallery does the rest.

A poster image (set through `data-poster`) is highly recommended.
While the user navigates the gallery, the poster is shown instead
of the video, so the gallery stays responsive. The video itself is
loaded only when the user clicks the poster. The thumbnail plugin
can also generate the thumbnails of YouTube, Vimeo and Dailymotion
videos automatically – set `loadYouTubeThumbnail: true` (or the
matching option for the other platforms) and pick a size with
`youTubeThumbSize`.

Videos are paused automatically when the user navigates to another
slide.

[role="mt-4"]
==== Plugin dependency

Online video slides require the *video* plugin
(`lg-video.min.js`). The plugin handles all four supported
platforms (YouTube, Vimeo, VK, Dailymotion) without any extra
configuration.
