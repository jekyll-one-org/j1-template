[role="dropcap"]
SkipAd embeds an excellent video player for YouTube based on VideoJS. All ads
on a YouTube video are blocked instantly for a great watching experience.
No more pre-roll, mid-roll, or overlay ads.

[role="mt-5"]
== Overview

The *J1 SkipAd* module is a client-side JavaScript adapter that enables
ad-free YouTube video playback within the J1 Template framework. It wraps
the link://videojs.com/[Video.js, {browser-window--new}] player library
with a custom YouTube playback technology ("`tech`") to embed and control
YouTube videos without exposing users to YouTube's advertising layer.

[role="mt-4"]
=== Key capabilities

Ad-free YouTube playback::
  Videos are loaded through the YouTube IFrame API via a VideoJS tech,
  bypassing YouTube's standard ad injection.

URL input & clipboard paste::
  Users paste or type a YouTube URL; the module validates it, extracts
  the video ID, and embeds the player.

Playlist management::
  A full-featured playlist system persists watched videos in
  `localStorage`, supporting list and card display modes, search,
  sorting, import/export, merge, per-entry CRUD operations, and
  automatic backup before clearing.

Responsive layout::
  The playlist automatically switches to single-column card mode on
  medium and smaller viewports (≤ 720 px) and restores the configured
  mode on wider screens.

VideoJS plugin integration::
  Configurable support for playback rate controls, skip buttons, zoom
  buttons, hotkeys, and auto-captions.

Video metadata extraction::
  Title, author, and duration are extracted at runtime from the YouTube
  player object and displayed in the UI. The YouTube IFrame API only
  populates the `author` field after playback begins, so a second
  extraction is performed on the `PLAYING` state change.

Inter-module communication::
  A message handler allows other J1 modules to communicate with SkipAd.

=== Configuration merging

At build time, the Liquid template merges both files:

{% raw %}
[source, liquid, role="noclip"]
----
{% assign skipad_defaults = modules.defaults.skipad.defaults %}
{% assign skipad_settings = modules.skipad.settings %}
{% assign skipad_options  = skipad_defaults | merge: skipad_settings %}
----
{% endraw %}

At runtime this is further deep-merged with jQuery:

[source, js, role="noclip"]
----
skipAdDefaults = $.extend({}, {{ skipad_defaults | replace: 'nil', 'null' | replace: '=>', ':' }});
skipAdSettings = $.extend({}, {{ skipad_settings | replace: 'nil', 'null' | replace: '=>', ':' }});
skipAdOptions  = $.extend(true, {}, skipAdDefaults, skipAdSettings);
----

The merged options are also stored on the adapter for external access:

[source, js, role="noclip"]
----
j1.adapter.skipad.skipAdOptions = skipAdOptions;
j1.adapter.skipad.ytVideoData   = {};
----

== Usage Hints

[role="mt-4"]
=== Embedding a video

. Navigate to the page containing the SkipAd player.
. Paste a YouTube URL into the input field using the *Paste* button or
  kbd:[Ctrl+V].
. Press kbd:[Enter] or click the *Load* button.
. The module extracts the video ID, creates a VideoJS player, and begins
  playback.

[role="mt-4"]
=== Playlist management

When the playlist is enabled (`playlist.enabled: true`), every played
video is automatically recorded. The playlist panel appears below the
player and supports:

Play:: Click the play overlay on any playlist thumbnail to replay the video.
Delete:: Click the trash icon to remove a single entry.
Search:: When search is enabled, filter entries by title or author.
Sort:: When sorting is enabled, reorder entries by various criteria.
Import/Export:: Use the playlist I/O handler to import from or export to
  JSON files.
Clear with backup:: Clearing the playlist first triggers a downloadable
  backup file (`skipAd-playlist-backup.json`) before removal.
Mode switching:: Toggle between list and card display modes at runtime;
  the selection persists in `localStorage` under the key `playlistMode`.
Merge:: Import playlist data and merge it with existing entries rather
  than replacing them.

[role="mt-4"]
=== Display modes

List mode:: A vertical list of rows, each containing a thumbnail with
  play overlay, duration badge, title, URL, relative time, and a delete
  button. CSS class: `playlist list-mode`.

Card mode:: A responsive grid of cards. The number of columns is
  controlled by `playlist.cards.perRow` (default: 4). CSS class:
  `playlist card-mode`. The grid is set via inline style
  `grid-template-columns: repeat(N, 1fr)`.

[role="mt-4"]
=== Responsive behavior

The module monitors the CSS media query `(max-width: 720px)` via
`matchMedia`. When the viewport is at or below this breakpoint:

* The playlist is forced into single-column card mode
  (`grid-template-columns: repeat(1, 1fr)`).
* The mode-switch control is hidden.

When the viewport exceeds the breakpoint the stored or configured mode is
restored, and the mode-switch control reappears.

[role="mt-4"]
=== Duplicate detection

If the same video ID is already loaded in the player, the module will
not create a new player instance. In the playlist, duplicate entries are
moved to the front (most-recent) rather than duplicated.

== Function Reference

[role="mt-4"]
=== Public API (Adapter)

The following methods and properties are exposed on `j1.adapter.skipad`:

[role="mt-4"]
[[public-api-init]]
==== init

Module initializer. Called by the J1 framework during page startup.

`Usage`: `init(options)`

.Behavior
. Initializes the `log4javascript` logger.
. Deep-merges default and user configuration into `skipAdOptions`.
. Stores `skipAdOptions` and an empty `ytVideoData` object on the adapter
  for external access.
. Registers a listener for the `ytVideoDataResolved` custom DOM event
  dispatched by the YouTube tech (`youtube.js`) and caches the received
  video data.
. Polls for page readiness (`j1.getState() === 'finished'` and
  `#content` visible) at `INIT_POLL_INTERVAL` (10 ms).
. Once ready:
.. Creates the `inputWrapperHandler` instance.
.. Reads the stored `playlistMode` from `localStorage` (fallback:
   configured `playlist.type`).
.. Creates the `#playlistHistory` DOM element with the appropriate CSS
   class (`list-mode` or `card-mode`) and grid template.
.. Renders the initial playlist via `skipAd.playlistManager.renderPlaylist()`
   or `skipAd.playlistManager.renderCards()`.
.. Sets up responsive breakpoint monitoring via `matchMedia('(max-width: 720px)')`.
.. Initializes event delegation handlers for delete and play actions.
.. Initializes the `playlistIOHandler` (import/export/clear) and wraps
   its `handleClear` method to create a backup before clearing.
.. Conditionally initializes `playlistSearchHandler` and
   `playlistSortHandler` based on configuration.
.. Initializes `playlistModeSwitchHandler` and
   `playlistMergeSwitchHandler`.
.. Sets module state to `finished` and logs initialization time.

[role="mt-4"]
==== backupPlaylist

Create a backup of the current playlist as a downloadable JSON file
(`skipAd-playlist-backup.json`) before deletion. This is automatically
called when `playlistIOHandler.handleClear()` is invoked.

`Usage`: `backupPlaylist()`

Returns:: `boolean` — `true` on success, `false` if no data or on error.

.Behavior
. Reads the `playlist` key from `localStorage`.
. Returns `false` if the data is missing or an empty array.
. Wraps the data in an object with a `backup_date` (ISO 8601 timestamp)
  and the `playlist` array.
. Creates a Blob download link and triggers the browser download.
. Revokes the object URL after download.

[role="mt-4"]

The `skipAd.playlistSortHandler` provides sorting functionality for
the playlist. Enabled only when `playlist.sort.enabled` is `true`.

[role="mt-4"]

The `skipAd.playlistModeSwitchHandler` handles toggling between list
and card display modes. The selected mode is persisted to `localStorage`
under the key `playlistMode`.

[role="mt-4"]
==== playlistMergeSwitchHandler

The `skipAd.playlistMergeSwitchHandler` controls whether imported
playlist data is merged with or replaces the existing playlist data.


[role="mt-4"]
=== playlistManager

The `skipAd.playlistManager` is an external singleton object responsible
for playlist rendering and event delegation. The adapter calls the
following methods during initialization and at runtime:

.playlistManager methods
[cols="2,4", width="100%", options="header", role="table-striped rtable mt-4"]
|===
| Method | Description

| `renderPlaylist()`
| Render the playlist in list mode — a vertical list of rows, each

  relative time, and a delete button.

| `renderCards()`
| Render the playlist in card mode — a CSS grid of cards, each
  containing a thumbnail with play overlay, duration badge, title,
  relative time, share button, and a delete button.

| `initDeleteHandler()`
| Set up event delegation on the `#playlistHistory` container. Clicks
  on any delete button trigger deletion of the corresponding entry.

| `initPlayHandler()`
| Set up event delegation on the `#playlistHistory` container. Clicks
  on any play overlay trigger playback of the corresponding video.
|===


[role="mt-5"]
== YouTube Tech (youtube.js)

The `youtube.js` file implements a VideoJS *Tech* (playback technology)
class that bridges VideoJS and the YouTube IFrame Player API.

Version:: 3.1.7 for J1 Template

[role="mt-4"]
=== Class: Youtube extends Tech

Registered via `videojs.registerTech('Youtube', Youtube)`.

[role="mt-4"]
==== Constructor

. Sets the poster image.
. Calls `setSrc()` with the initial source.
. Adds CSS classes (`vjs-youtube`, `vjs-youtube-mobile`).
. If the YouTube API is ready, calls `initYTPlayer()` immediately;
  otherwise queues the instance in `Youtube.apiReadyQueue`.

[role="mt-4"]
==== Key methods

.Key methods
[cols="2,4", width="100%", options="header", role="table-striped rtable mt-4"]
|===
| Method | Description

| `initYTPlayer()`
| Creates a `YT.Player` instance with merged player vars from
  configuration. Binds all YouTube event callbacks (`onReady`,
  `onPlaybackQualityChange`, `onPlaybackRateChange`,
  `onStateChange`, `onVolumeChange`, `onError`). Supports
  `enablePrivacyEnhancedMode` for `youtube-nocookie.com`.

| `onPlayerReady()`
| Fires on YT player ready. Extracts video metadata via
  `getVideoData()`, caches it, and dispatches the
  `ytVideoDataResolved` CustomEvent on `document` with source
  `'onPlayerReady'`. Triggers `triggerReady()` on the tech.

| `onPlayerStateChange(e)`
| Maps YouTube state codes to VideoJS events. On `PLAYING` state,
  re-reads video data to obtain the `author` field (only available
  after playback begins) and dispatches `ytVideoDataResolved` again
  with source `'onPlayerStateChange:PLAYING'` if data changed.

| `onPlayerError(e)`
| Stores error number and triggers `pause` + `error` events.

| `videoData()`
| Public getter returning the full `getVideoData()` object from the YT
  player. Returns cached data or performs a live read.

| `videoTitle()`
| Backward-compatible getter returning the video title string.

| `play()`
| Calls `ytPlayer.playVideo()` or loads a new video if the ID changed.
  Handles playlist-based playback when `url.listId` is present.

| `pause()`
| Calls `ytPlayer.pauseVideo()`.

| `currentTime()` / `setCurrentTime(s)`
| Get/set playback position. Seeking during pause uses an interval-based
  workaround (polls every 250 ms until `currentTime` changes).

| `duration()`
| Returns `ytPlayer.getDuration()` with error protection.

| `volume()` / `setVolume(v)`
| Get/set volume (0–1 range, mapped to YT's 0–100).

| `muted()` / `setMuted(m)`
| Get/set mute state.

| `playbackRate()` / `setPlaybackRate(r)`
| Get/set playback speed with error handling (falls back to rate 1).

| `buffered()` / `seekable()`
| Return time ranges based on YT player state.

| `networkState()` / `readyState()`
| Map YT player states to HTML5 media element constants.

| `src(s)` / `setSrc(source)`
| Parse URL via `Youtube.parseUrl()` and configure the player.

| `dispose()`
| Stops video, destroys YT player, cleans up DOM and CSS classes.

| `checkHighResPoster()`
| Attempts to load `maxresdefault.jpg` for the video; falls back
  silently if the high-resolution poster is unavailable.

== Event Management

The SkipAd module uses a multi-layered event system spanning YouTube's
native API, the VideoJS tech, and custom DOM events.

[role="mt-4"]
=== YouTube IFrame API → VideoJS events (Tech layer)

The YouTube tech (`youtube.js`) translates YouTube player state
codes into VideoJS events inside `onPlayerStateChange()`:

.Tech events
[cols="1,2,3", width="100%", options="header", role="table-striped rtable mt-4"]
|===
| YT State | YT Constant | VideoJS Events Triggered

| `-1`
| (unstarted)
| `loadstart`, `loadedmetadata`, `durationchange`, `ratechange`

| `0`
| `YT.PlayerState.ENDED`
| `ended`

| `1`
| `YT.PlayerState.PLAYING`
| `timeupdate`, `durationchange`, `playing`, `play`

| `2`
| `YT.PlayerState.PAUSED`
| `canplay`, `pause` (or `seeked` if seeking)

| `3`
| `YT.PlayerState.BUFFERING`
| `timeupdate`, `waiting`
|===

[role="mt-4"]
=== Custom DOM events (metadata resolution)

The YouTube tech dispatches a `ytVideoDataResolved` CustomEvent on
the `document` object at two points:

.ytVideoDataResolved dispatch points
[cols="2,3,2", width="100%", options="header", role="table-striped rtable mt-4"]
|===
| Source | Trigger | Notes

| `onPlayerReady()`
| YT player becomes ready
| Initial extraction; `author` field is typically empty

| `onPlayerStateChange(PLAYING)`
| Playback begins
| Re-extraction; `author` field is now populated by the YT API
|===

.ytVideoDataResolved event detail
[cols="2,1,4", width="100%", options="header", role="table-striped rtable mt-4"]
|===
| Field | Type | Description

| `videoData`
| `Object`
| Full object from `YT.Player.getVideoData()` (includes `video_id`,
  `title`, `author`, `video_quality`, etc.)

| `videoTitle`
| `string`
| The video title string (convenience field)

| `source`
| `string`
| Identifier of the dispatch point: `'onPlayerReady'` or
  `'onPlayerStateChange:PLAYING'`
|===

The adapter (`skipad.js`) listens for this event during `init()`:

[source, js, role="noclip"]
----
document.addEventListener('ytVideoDataResolved', (e) => {
  if (e.detail && e.detail.videoData) {
    j1.adapter.skipad.ytVideoData = e.detail.videoData;
  }
});
----

The `videoData()` method returns the object from

[source, json]

NOTE: The `author` field is only populated by the YouTube IFrame API
after playback begins. The initial call to `getVideoData()` in
`onPlayerReady()` will return an empty string for `author`.
