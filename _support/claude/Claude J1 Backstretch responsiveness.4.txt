**1. ResizeObserver instead of window.resize** (line 686, constructor)
SwiperJS's `Resize` module observes the *container element itself* via `ResizeObserver`, not just the window. This detects container-level size changes caused by CSS transitions, flex/grid reflows, or parent resizing that `window.resize` never fires for. A fallback to `window.resize` is kept for older browsers.

**2. requestAnimationFrame batching** (line 686, constructor)
SwiperJS wraps resize callbacks in `requestAnimationFrame` to coalesce rapid-fire events into a single paint cycle. Backstretch previously called `resize()` synchronously on every event, causing layout thrashing during continuous window dragging.
