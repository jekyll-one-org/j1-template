**Root Cause:** The #5 fix incorrectly borrowed SwiperJS's `updateSize()` pattern of subtracting CSS padding from `clientWidth`/`clientHeight`. This works for SwiperJS because its slides are *content-flow children* inside the content box. But backstretch's `.backstretch` wrap is *absolutely positioned* with `width: 100%; height: 100%` — per CSS spec, those percentages resolve against the containing block's **padding box**, not content box. Since `.attic` defines its visual height entirely through `padding-top: 600px`, subtracting that padding yielded `boxHeight = 0`, preventing images from rendering.

**Fixes in `backstretch.js`:**

1. **Box dimension calculation** — Removed the padding subtraction. Now reads `clientWidth`/`clientHeight` directly (which equals content + padding, matching the absolutely-positioned wrap's resolved dimensions). Added a fallback chain: `clientWidth → offsetWidth → jQuery.width()`.

2. **Cover mode rounding** — Changed `Math.round` → `Math.ceil` for the computed dimension. Rounding down by even 1px can leave a visible gap at the container edge where the background color bleeds through.

3. **Fit mode rounding** — Changed `Math.round` → `Math.floor`. Rounding up by 1px can push the image outside the box boundary, triggering scrollbars or clipping.

4. **`maxHeight: 'none'`** — Added alongside the existing `maxWidth: 'none'` on item styles. In cover mode the image height can exceed the container; an inherited `max-height` rule would clip it.

5. **`$itemWrapper` guard** — Added a null/empty check before reading natural dimensions. ResizeObserver can fire before `show()` has created the first item wrapper.

6. **Enhanced debug logging** — Now logs natural dimensions, aspect ratio, and scale mode alongside box dimensions for easier troubleshooting.

**Fixes in `attic.js`:**

7. **Window load resize** — Added a `$(window).one('load')` handler that triggers a final `backstretch('resize')` after all page resources (fonts, images, deferred CSS) have loaded, catching late reflows that the ResizeObserver may miss during initial render.
