Für VideoJS Player können Sie Chapters und Captions dynamisch zur Laufzeit
über die VideoJS API hinzufügen.

Hier ist eine vollständige Lösung für das dynamische Hinzufügen von
Chapters und Captions zu VideoJS Playern.

## Hauptmethoden für dynamische Tracks:

### 1. **addRemoteTextTrack()**

Tracks hinzufügen.

```javascript
videojsPlayer.addRemoteTextTrack({
    default:  true,
    kind:     'chapters',      // oder 'captions', 'subtitles'
    src:      '/assets/video/tracks/html5/peck_pocketed/chapters/video.vtt',
    srclang:  'en',
    label:    'English'
    
}, false);
```

## WEBVTT Datei-Beispiele
--------------------------------------------------------------------------------

### Chapters (chapters.vtt):
```
WEBVTT

CHAPTER
00:00:00.000 --> 00:01:30.000
Intro

CHAPTER  
00:01:30.000 --> 00:03:45.000
Hauptteil
```

### Captions (captions-de.vtt):
```
WEBVTT

1
00:00:00.000 --> 00:00:03.000
Willkommen zu diesem Video!

2
00:00:03.000 --> 00:00:06.000
Heute zeigen wir Ihnen VideoJS.
```

## Wichtige Punkte:

- **Blob URLs**: Für dynamische Inhalte können Sie WEBVTT-Strings in
  Blob URLs umwandeln
- **Track Management**: Entfernen Sie existierende Tracks bevor Sie
  neue hinzufügen
- **Multiple Sprachen**: Unterschiedliche `srclang` Werte für
   verschiedene Sprachen
- **Default Track**: `default: true` aktiviert den Track automatisch

Buttons zum Testen der verschiedenen Funktionen.

Um Kapitel (Chapters) und Untertitel (Captions) in VideoJS dynamisch zur
Laufzeit hinzuzufügen oder zu entfernen, sind WebVTT-Dateien und
JavaScript-Methoden erforderlich.

Hier ist eine detaillierte Anleitung mit Beispielen:

### **1. WEBVTT-Dateien für Chapters und Captions**  

#### Kapitel (Chapters)-Beispiel:  
```webvtt
WEBVTT

00:00:00.000 --> 00:00:30.000
Einführung
00:00:30.000 --> 00:01:15.000

Hauptteil
00:01:15.000 --> 00:02:00.000

Zusammenfassung:

```
- **Struktur**: Zeitstempel im Format `hh:mm:ss.mss`, gefolgt vom Kapiteltitel
  (jeweils nur eine Zeile).  
- **Hinweis**: Kapitel müssen lückenlos aufeinanderfolgen[2][3].  

#### Untertitel (Captions)-Beispiel:  
```webvtt
WEBVTT

00:00:05.000 --> 00:00:10.000
Hallo, willkommen bei dieser Demo.

00:00:15.000 --> 00:00:20.000
Dies ist ein dynamischer Untertitel.
```

### **2. JavaScript für dynamische Tracks** 

#### Kapitel oder Untertitel hinzufügen:

```javascript
const player = videojs('my-video');

// Neuen Track erstellen
const track = player.addTextTrack('chapters', 'Deutsch', 'de'); // Für Captions: 'captions'

// Cues aus WebVTT-Datei laden (Beispiel mit Kapiteln)
track.addCue(new VTTCue(0, 30, 'Einführung')); // Start, Ende, Text
track.addCue(new VTTCue(30, 75, 'Hauptteil'));
track.mode = 'showing'; // Track sichtbar machen
```

- **Erklärung**:  
  - `addTextTrack(kind, label, language)` erstellt einen neuen Track.  
  - `VTTCue(start, end, text)` definiert Zeitabschnitte und Text[1][4].  
  - Für Remote-WebVTT-Dateien:  
    ```javascript
    const trackEl = document.createElement('track');
    trackEl.kind = 'chapters';
    trackEl.label = 'Deutsch';
    trackEl.src = 'pfad/zur/datei.vtt';
    player.remoteTextTracks().addTrackElement(trackEl);
    ```

#### Tracks entfernen:

```javascript
const tracks = player.remoteTextTracks();

// Track deaktivieren und aus DOM entfernen
tracks.tracks().forEach((track, index) => {
  if (track.label === 'Deutsch') {
    track.mode = 'disabled'; // Deaktiviert den Track
    const trackEl = track.trackElement;
    if (trackEl) trackEl.parentNode.removeChild(trackEl); // Entfernt DOM-Element
  }
});
```

**Hinweis**: Direktes Löschen via API ist nicht möglich.
Zum Löschen is ein  Deaktivieren + DOM-Manipulation sind notwendig[5][6].  

---

### **Zusammenfassung der Schritte**

1. **WebVTT-Dateien erstellen**:  
   - Kapitel: Zeitbereiche + Titel (eine Zeile pro Kapitel).  
   - Untertitel: Zeitbereiche + Text.  

2. **Tracks hinzufügen**:  
   - `addTextTrack()` für manuelle Cues.  
   - ``-Element für Remote-Dateien.  

3. **Tracks entfernen**:  
   - Track deaktivieren (`mode = "disabled"`).  
   - DOM-Element entfernen (falls vorhanden).  

> **Wichtig**: Bei Verwendung von Remote-Dateien muss der Track über ein
``-Element hinzugefügt werden, um ihn später entfernen zu können[5][6].
Native Browser-APIs erlauben kein direktes Löschen von Tracks, daher ist
eine DOM-Manipulation nötig.

Welche WebVTT-Struktur ist notwendig, um Kapitel und Captions im Web
zu ergänzen?

Perplexity
--------------------------------------------------------------------------------

Um Kapitel (Chapters) und Untertitel (Captions) im Web über WebVTT zu
ergänzen, ist jeweils eine spezifische Struktur der .vtt-Datei notwendig.
Beide Formate basieren auf dem gleichen Grundprinzip, unterscheiden sich
aber in der Verwendung und im Aufbau der Einträge.

## **Grundstruktur einer WebVTT-Datei**

- Die Datei beginnt immer mit dem Header `WEBVTT` (optional gefolgt
  von einem Kommentar oder Titel).
- Es folgen die einzelnen Einträge (Cues), die jeweils mit einem
  Zeitstempel beginnen.
- Zeitstempel-Format: `hh:mm:ss.mmm --> hh:mm:ss.mmm`
- Zwischen jedem Eintrag steht eine Leerzeile.
- Die Dateiendung muss `.vtt` sein[1][2][3].

---

## **Beispiel: WebVTT für Kapitel (Chapters)**

```vtt
WEBVTT

00:00:00.000 --> 00:00:30.000
Einführung

00:00:30.000 --> 00:01:15.000
Hauptteil

00:01:15.000 --> 00:02:00.000
Zusammenfassung
```

- Jeder Cue markiert den Start und das Ende eines Kapitels.
- Der Text unterhalb des Zeitstempels ist der Kapitelname[4][5][1].
- Keine weiteren Formatierungen oder Einstellungen notwendig.

---

## **Beispiel: WebVTT für Untertitel (Captions)**

```vtt
WEBVTT

00:00:05.000 --> 00:00:10.000
Hallo, willkommen bei dieser Demo.

00:00:15.000 --> 00:00:20.000
Dies ist ein dynamischer Untertitel.
```

- Jeder Cue enthält den gesprochenen Text für den jeweiligen Zeitraum.
- Optional KANN jeder Cue eine fortlaufende Nummer erhalten, ist aber
  nicht zwingend[2][1].
- Für Captions können zusätzliche Einstellungen wie Position oder Stil
  nach dem Zeitstempel ergänzt werden (z.B. `line:90%`)[2].

---

## **Wichtige Hinweise zur Struktur**

- Die Zeitstempel müssen exakt formatiert sein: `Stunden:Minuten:Sekunden.Millisekunden`[1][2].
- Zwischen den Einträgen muss jeweils eine Leerzeile stehen[1][2].
- Die Datei muss als reiner Text gespeichert werden (kein Rich Text)[4][5].
- Für Kapitel und Untertitel wird die gleiche Grundstruktur verwendet,
  aber die Interpretation durch den Player erfolgt über das `kind`-Attribut
  im ``-Element (z.B. `kind="chapters"` oder `kind="captions"`)[3].

---

## **Zusammenfassung**

- **Kapitel:** Jeder Cue steht für ein Kapitel, der Text ist der Kapitelname.
- **Captions:** Jeder Cue steht für einen Untertitel, der Text ist der
  anzuzeigende Untertitel.
- **Technische Anforderungen:** Korrekte Zeitstempel, Leerzeilen,
  Dateiendung `.vtt`, reiner Text[1][2][4].

Mit dieser Struktur können Sie sowohl Kapitel als auch Untertitel einfach
im Web ergänzen.

Wie kann man bei einem VideoJS Player Caption-Tracks zur Laufzeit manuell
per JavaScript ein- und ausschalten?

Perplexity
--------------------------------------------------------------------------------

## Caption-Tracks bei VideoJS zur Laufzeit per JavaScript ein- und ausschalten

Um Caption-Tracks (z. B. Untertitel oder Übersetzungen) in einem VideoJS
Player zur Laufzeit manuell per JavaScript ein- oder auszuschalten, kannst
du direkt auf die textTracks()-API des Players zugreifen und dort den mode
der gewünschten Spur setzen.

**Vorgehen:**

- Hole dir die TextTracks des Players mit `player.textTracks()`.
- Durchlaufe die Tracks und setze für den gewünschten Track die
  Eigenschaft `mode` auf `"showing"` (einblenden) oder `"disabled"`
  (ausblenden).

**Beispiel:**

```javascript
// Angenommen, 'player' ist deine VideoJS-Instanz
var tracks = player.textTracks();

for (var i = 0; i < tracks.length; i++) {
  var track = tracks[i];
  // Beispiel: Englische Captions anzeigen
  if (track.kind === 'captions' && track.language === 'en') {
    track.mode = 'showing'; // Einschalten
  } else {
    track.mode = 'disabled'; // Andere ausblenden
  }
}
```

Mit dieser Methode kannst du gezielt einzelne Tracks ein- oder ausschalten,
ohne dass der Nutzer das Menü benutzen muss[6].

**Hinweise:**

- `"showing"`: Track wird angezeigt.
- `"disabled"`: Track ist deaktiviert und wird nicht angezeigt.
- `"hidden"`: Track ist aktiv, aber nicht sichtbar (z. B. für Metadaten).

Weitere Details findest du in der offiziellen VideoJS-Dokumentation zum 
hema Text Tracks[6].

**Zusatz:**

Falls du die Caption-Settings im Menü ausblenden möchtest, kannst du dies
über die Player-Option `textTrackSettings: false` und ein wenig CSS
erreichen:

```javascript
var player = videojs('myvideo', {
  textTrackSettings: false
});
```
```css
.vjs-texttrack-settings {
  display: none;
}
```

Damit wird das Caption-Settings-Menü entfernt, hat aber keinen Einfluss auf
das manuelle Umschalten der Tracks per JavaScript[6][1][2].
